# Den-native agent guidance

Den can resolve agent guidance from first-class guidance entries instead of
asking agents to manually discover important documents in `_global` or treating
an autogenerated `AGENTS.md` snapshot as the live source of truth.

## Model

A guidance entry points at an existing Den document:

- `project_id` — the guidance scope. Use `_global` for guidance inherited by all
  projects; use a normal project id for project-local guidance.
- `document_project_id` and `document_slug` — the referenced Den document.
- `importance` — `required` or `important`.
- `audience` — optional labels such as `pi`, `conductor`, `coder`, or
  `reviewer`.
- `sort_order` — deterministic ordering within the scope; lower values come
  first.
- `notes` — optional operator context explaining why the document is included.

Resolution is deterministic:

1. `_global` guidance entries.
2. Project-local guidance entries.
3. `sort_order` ascending.
4. `required` before `important` for ties.
5. Referenced document project/slug and entry id for stable tie-breaking.

The resolved response includes rendered Markdown plus source metadata for every
included document, so consumers can display provenance and operators can identify
where policy changes belong.

## API

Resolve guidance for a project:

```http
GET /api/projects/{projectId}/agent-guidance
```

List entries:

```http
GET /api/projects/{projectId}/agent-guidance/entries?includeGlobal=true
```

Add or update an entry:

```http
POST /api/projects/{projectId}/agent-guidance/entries
Content-Type: application/json

{
  "document_project_id": "_global",
  "document_slug": "pi-conductor-guidance-default",
  "importance": "required",
  "audience": ["pi", "conductor"],
  "sort_order": 10,
  "notes": "Default conductor operating policy"
}
```

Delete an entry:

```http
DELETE /api/projects/{projectId}/agent-guidance/entries/{entryId}
```

## MCP tools

- `get_agent_guidance(project_id)` resolves the packet.
- `list_agent_guidance_entries(project_id, include_global)` lists entries.
- `add_agent_guidance_entry(...)` adds or updates an entry; its `audience` argument is a comma-separated string such as `pi,conductor`.
- `delete_agent_guidance_entry(project_id, entry_id)` removes an entry scoped to the owning project.

## CLI

```bash
# Print the resolved Markdown packet for the current directory's project.
den guidance

# Include inherited _global entries while listing project entries.
den guidance list --project den-mcp --include-global

# Mark a global document as required guidance inherited by all projects.
den guidance add pi-conductor-guidance-default \
  --project _global \
  --doc-project _global \
  --importance required \
  --audience pi,conductor \
  --order 10

# Mark a project-local document as important guidance.
den guidance add project-policy --project den-mcp --importance important --order 100
```

## Pi integration

The tracked Pi Den extension loads resolved guidance on `session_start`, which
also runs during `/reload`. If guidance entries are configured, it appends the
resolved Markdown packet to Pi's system prompt for each agent turn. The
`/den-agent-guidance` command refreshes and displays the current packet.

The extension treats guidance resolution as additive: if the server is older or
no entries are configured, Pi continues with the normal context files and skills.

## Bootstrap `AGENTS.md` strategy

During migration, keep generated `AGENTS.md` snapshots as compatibility exports
for tools that do not yet know how to ask Den for guidance. A generic bootstrap
file should tell agents:

1. Connect to Den using the configured MCP server or local CLI.
2. Resolve current guidance with `get_agent_guidance` or `den guidance` for the
   active project.
3. Treat the resolved Den packet and referenced Den documents as the live source
   of truth.
4. Do not edit generated or bootstrap `AGENTS.md` files for policy changes.
   Update the Den documents or guidance entries instead.
5. Regenerate/export compatibility snapshots only after the Den guidance source
   is updated.

The existing `scripts/generate_agents.py` workflow remains supported while the
Den-native path proves itself.

## Editing workflow

For policy edits:

1. Edit the referenced Den document (or create a new Den document).
2. Add/update the guidance entry if inclusion, importance, audience, or order
   changes.
3. Resolve guidance and verify the source list.
4. Reload Pi or run `/den-agent-guidance` to refresh the injected packet.
5. Regenerate compatibility `AGENTS.md` snapshots only when needed for tools that
   still rely on them.
